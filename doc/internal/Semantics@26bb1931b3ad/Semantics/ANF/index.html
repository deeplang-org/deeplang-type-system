data_kind is used to classify different kinds of aggregate data. In the ANF IR, different kinds of aggregate data share a uniform representation.

type lvalue = {
var : variable;
path : path;
}

lvalue is a value that may appear at the left hand side of an assignment. So every lvalue corresponds to a memory address. Some examples: point.x, *ptr

and path = path_node list
and path_node =
| Field of int
| Deref

path_node is used to select a part of an existing lvalue:

  • Field k selects the k-th field from an aggregate, i.e. a tuple, a struct, an ADT or a method dictionary
  • Deref selects the address a pointer points to

A value in the ANF IR is something immediately available without needing any computation.

type program =
| Return of span * expr
| Jump of span * block_label * value list
| Stmt of span * statement * program
| Branch of branching
| DefBlock of block_definition * program
  • Return(span, expr) returns the result of expr as the result of the whole program. If expr is a function application, then this represents a tail-call
  • Jump(span, label, args) jumps to the block with name label with args
  • Stmt(span, stmt, body) first executes stmt and then executes body. span is the source location of stmt
  • Branch br is a unified construction for all control flow branching structures, e.g. if/else and pattern matching
  • DefBlock(def, body) defines a new block with def, and executes body
and branching = {
br_src : span;
br_matched : value;
br_branches : (adt_label * program) list;
br_default : program option;
}

branching is a simple switch on ADT label (integer tag). Each ADT label has an associated branch in br_branches, and there is an optional default branch br_default.

branching is used to represent all control flow branching in the source language:

  • if/else is translated to pattern matching on boolean
  • pattern matching is decomposed to nested branching
